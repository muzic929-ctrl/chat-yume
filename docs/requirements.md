# 需求规格说明

## 1. 项目背景
一款手机网页版 AI 聊天应用，用户可创建虚拟聊天角色，与 AI 角色进行符合其人设的对话。

## 2. 用户角色
- 终端用户：使用应用创建角色、聊天的普通用户

## 3. 功能需求

### 3.1 用户系统
- FR-01：用户可使用用户名和密码注册账号
- FR-02：已注册用户可登录进入应用
- FR-03：登录状态保持（关闭浏览器后仍保持登录）
- FR-04：不同用户的数据完全隔离

### 3.2 角色管理
- FR-05：用户可创建 AI 聊天角色，填写以下信息：
  - 姓名（必填）
  - 性别（必填，男/女/其他）
  - 年龄（必填）
  - 身高 cm（必填）
  - 体重 kg（必填）
  - 性格描述（必填）
  - 备注（可选）
- FR-06：角色头像可从预设头像中选择，或从手机相册上传照片
- FR-07：上传的头像自动裁剪为正方形
- FR-08：用户可编辑已创建角色的信息
- FR-09：用户可删除已创建的角色（角色删除后聊天记录一并删除）
- FR-10：角色列表展示所有已创建角色（卡片形式）

### 3.3 聊天功能
- FR-11：用户可与角色进行文字对话
- FR-12：AI 回复内容需根据角色的人设信息生成（免费模拟模式）
- FR-13：聊天界面采用聊天气泡风格（用户右侧、AI 左侧）
- FR-14：聊天记录持久化保存

### 3.4 真 AI 模式（可选）
- FR-15：用户可在设置中切换为真 AI 模式
- FR-16：支持配置以下模型 API：
  - OpenAI (ChatGPT)
  - Anthropic (Claude)
  - DeepSeek
  - 通义千问（阿里）
  - 智谱 ChatGLM
  - 月之暗面 Kimi
  - 豆包（字节）
  - 文心一言（百度）
  - 自定义（兼容 OpenAI 接口）
- FR-17：用户自行填入 API Key，存在本地
- FR-18：提供"测试连接"按钮验证配置

### 3.5 个性化
- FR-19：用户可修改角色的备注名称
- FR-20：用户可切换聊天背景（随机预设颜色 + 从相册上传图片）
- FR-21：用户可清空聊天记录

## 4. 非功能需求
- NFR-01：手机端优先，适配 375px-414px 宽度
- NFR-02：主色调为白色，界面简洁
- NFR-03：无需后端服务器，纯前端运行
- NFR-04：所有数据存储在浏览器 localStorage

## 5. 不做什么（范围边界）
- 不做语音/图片消息
- 不做消息撤回/转发
- 不做群聊
- 不做好友系统
- 不做消息推送通知
